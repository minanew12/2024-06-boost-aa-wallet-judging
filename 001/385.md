# `claimIncentiveFor` Function Allowing Users to Claim Referral Fees

## Summary

## Vulnerability Detail

The `claimIncentiveFor` function in the `BoostCore.sol` contract allows users to specify a `referrer_` address without any verification. This means that users can potentially set the `referrer_` address to their own address and claim the referral share of the claim fee.

```solidity
function claimIncentiveFor(
        uint256 boostId_,
        uint256 incentiveId_,
        address referrer_,//@audit-no verification
        bytes calldata data_,
        address claimant
    ) public payable nonReentrant {
        BoostLib.Boost storage boost = _boosts[boostId_];
        if (msg.value < claimFee) revert BoostError.InsufficientFunds(address(0), msg.value, claimFee);
        _routeClaimFee(boost, referrer_);

        // wake-disable-next-line reentrancy (false positive, function is nonReentrant)
        if (!boost.validator.validate(boostId_, incentiveId_, claimant, data_)) revert BoostError.Unauthorized();
        if (!boost.incentives[incentiveId_].claim(claimant, data_)) {
            revert BoostError.ClaimFailed(claimant, data_);
        }
    }
```
The _routeClaimFee function does not verify the referrer_ address:
```solidity
 function _routeClaimFee(BoostLib.Boost storage boost, address referrer_) internal {
        if (claimFee == 0) return;
        uint256 netFee = claimFee;

        // If a referrer is provided, transfer the revshare and reduce the net fee
        if (referrer_ != address(0)) {
            uint256 referralShare = claimFee * boost.referralFee / FEE_DENOMINATOR;
            netFee -= referralShare;
            referrer_.safeTransferETH(referralShare);
        }

        // The remaining fee is split between the owner and the protocol
        boost.owner.safeTransferETH(netFee / 2);
        protocolFeeReceiver.safeTransferETH(address(this).balance);
    }
```

## Impact
* users can claim referral fees
